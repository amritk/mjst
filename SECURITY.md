# Security Policy

## Supported versions

mjst is pre-alpha (0.x). Only the latest published version of each package is supported. Breaking changes can land in any 0.x release.

## Reporting a vulnerability

Please **do not** open a public issue for security problems.

Report vulnerabilities by opening a [private security advisory](https://github.com/amritk/mjst/security/advisories/new) on GitHub. Include:

- The affected package and version
- A description of the issue and its impact
- A minimal reproduction (schema + invocation) if possible

We aim to acknowledge reports within 7 days and to publish a fix or mitigation as soon as a patch is ready. Coordinated disclosure is appreciated.
